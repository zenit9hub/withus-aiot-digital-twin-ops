# 제어규칙

## 1. 문서 목적
본 문서는 `엣지제어운영서버`가 따르는 규칙 기반 제어 기준을 정의한다.  
목적은 온도, 진동, 냉각 제어, 셧다운 규칙을 한 장으로 고정해 구현과 설명의 기준으로 사용하는 데 있다.

## 2. 적용 범위
초기 MVP(Minimum Viable Product) 기준 적용 범위는 아래와 같다.

- 설비 1종: `conveyor-01`
- 룸 1개: `room-01`
- 센서 2종: `temp-01`, `vib-01`
- 제어 대상 2종: `conveyor-01`, `aircon-01`

## 3. 제어 원칙
- 규칙 기반 제어가 셧다운 여부를 최종 결정한다.
- AI(Artificial Intelligence)는 제어가 아니라 운영지원을 담당한다.
- 시뮬레이터는 환경 상태를 계산하고, 엣지제어운영서버는 상태를 해석한다.
- 냉각 실패 시나리오는 `응답 실패 판정`이 아니라 `냉각 제어 메시지 미전달`로 표현한다.

## 4. 온도 제어 규칙
### 4.1 상태 구간
- `45도 미만`: `NORMAL`
- `45도 이상 50도 미만`: `WARNING`
- `50도 이상`: `CRITICAL`

### 4.2 동작 규칙
- `WARNING` 진입 시 냉각 제어를 시도한다.
- `CRITICAL` 진입 시 즉시 셧다운하지 않고 지속 시간을 확인한다.
- `CRITICAL` 상태가 일정 시간 지속되면 셧다운을 수행한다.

### 4.3 권장 초기 데모값
- `WARNING threshold = 45도`
- `CRITICAL threshold = 50도`
- `CRITICAL duration = 3 tick` 또는 `약 5~10초`

## 5. 진동 제어 규칙
### 5.1 상태 구간
- `0.70 미만`: `NORMAL`
- `0.70 이상 0.85 미만`: `VIBRATION_WARNING`
- `0.85 이상`: `VIBRATION_CRITICAL`

### 5.2 동작 규칙
- `VIBRATION_WARNING`에서는 경고와 운영지원 표시를 수행한다.
- `VIBRATION_CRITICAL`에서는 즉시 셧다운을 수행한다.

### 5.3 권장 초기 데모값
- `WARNING threshold = 0.70`
- `CRITICAL threshold = 0.85`

## 6. 냉각 제어 규칙
### 6.1 정상 시나리오
- 온도 `WARNING` 진입 시 `aircon/control`로 `ON` 메시지를 발행한다.
- 시뮬레이터가 냉각 효과를 반영한다.
- 온도가 정상 범위로 회복되면 `OFF` 메시지를 발행할 수 있다.

### 6.2 냉각 실패 시나리오
- 온도 `WARNING` 진입 시 냉각 제어를 시도한다.
- 냉각 실패 시나리오에서는 시뮬레이터로 제어 메시지가 전송되지 않으므로 냉각 효과가 반영되지 않는다.
- 결과적으로 온도는 계속 유지되거나 상승한다.
- `CRITICAL` 상태가 지속되면 셧다운을 수행한다.

## 7. 셧다운 규칙
셧다운은 아래 두 조건에서만 수행한다.

- 진동이 `VIBRATION_CRITICAL` 구간에 진입한 경우
- 온도가 `CRITICAL` 구간에 진입한 뒤 지속 시간 조건을 만족한 경우

권장 `conveyor/control` 메시지:
- `controlAction = SHUTDOWN`
- `controlSource = RULE`
- `reason = temperature critical sustained` 또는 `reason = vibration critical`

## 8. 이벤트 생성 규칙
권장 이벤트는 아래와 같다.

- `WARNING`
- `RECOVERY`
- `COOLING_COMMAND_SENT`
- `COOLING_COMMAND_SKIPPED`
- `VIBRATION_WARNING`
- `VIBRATION_CRITICAL`
- `SHUTDOWN`

이벤트는 센서 입력값 그대로가 아니라, 상태 전이와 제어 결과를 기준으로 생성한다.

이벤트 중복 방지 원칙:
- 같은 상태가 유지되는 동안 동일 `WARNING` 이벤트를 반복 발행하지 않는다.
- `RECOVERY`는 회복 시점에 1회만 발행한다.
- `SHUTDOWN`은 `RESET` 전까지 1회만 발행한다.

## 9. 운영 대시보드 표시 기준
운영 대시보드에는 아래를 표시한다.

- 현재 온도
- 현재 진동
- 현재 설비 상태
- 최근 냉각 제어 명령 상태
- 최근 이벤트
- 규칙 기반 판단 결과
- AI 운영지원 결과

## 10. 고도화 항목
아래 항목은 초기 MVP 이후 확장 대상으로 본다.

- 에어컨 `status` 기반 응답 확인
- 응답 timeout 기반 실패 판정
- 냉각 시스템 가동 중 온도 비회복 경고
- 제어 실패 상세 메시지 자동 생성
