# Dashboard

## 1. 문서 목적
본 문서는 핵심본체 샘플 프로젝트에서 사용하는 운영 대시보드의 화면 구성 기준을 정의한다.  
목적은 센서 상태, 규칙 기반 제어 결과, AI(Artificial Intelligence) 운영지원 결과를 한 화면에서 빠르게 이해할 수 있도록 기준을 정리하는 데 있다.

## 2. 설계 원칙
- 운영자는 현재 상태를 `3초 이내`에 파악할 수 있어야 한다.
- 원시 데이터보다 `현재 상태`, `최근 제어`, `최근 이벤트`를 우선 표시한다.
- 규칙 기반 제어 결과와 AI 운영지원 결과는 구분해 보여준다.
- 초기 MVP(Minimum Viable Product)에서는 에어컨 실제 응답 상태가 아니라 `최근 냉각 제어 명령 상태`를 표시한다.
- 경고와 셧다운은 색상과 텍스트로 동시에 표현한다.

## 3. 화면 목적
운영 대시보드는 아래 질문에 바로 답할 수 있어야 한다.

- 지금 설비 상태는 정상인가, 경고인가, 셧다운인가
- 온도와 진동 중 무엇이 문제인가
- 최근 냉각 제어가 수행되었는가
- 최근 어떤 이벤트가 발생했는가
- 규칙 기반 제어는 어떤 결정을 내렸는가
- AI는 어떤 점검 또는 운영 가이드를 제시하는가

## 4. 권장 화면 구성
### 4.1 상단 요약 영역
상단에는 아래 카드 4개를 배치하는 것을 권장한다.

- `설비 상태`: `NORMAL`, `WARNING`, `SHUTDOWN`
- `현재 온도`: 실시간 수치와 단위
- `현재 진동`: 실시간 수치와 단위
- `최근 냉각 제어 명령 상태`: `ON`, `OFF`, `NOT_SENT`

### 4.2 중앙 상태 영역
중앙에는 아래 정보를 크게 표시한다.

- 현재 `runtimeState`
- 현재 `healthState`
- 현재 `alertLevel`
- 최근 규칙 기반 제어 결정

예:
- `WARNING / 냉각 제어 수행 중`
- `CRITICAL / 셧다운 대기`
- `SHUTDOWN / 규칙 기반 제어 완료`

### 4.3 센서 시계열 영역
온도와 진동은 별도 차트로 표시한다.

- 온도 차트: 최근 1~5분 추이
- 진동 차트: 최근 1~5분 추이
- 경고/임계 기준선 표시

권장 이유:
- 온도는 `상승`, `회복`, `비회복` 흐름을 설명하기 좋다.
- 진동은 `경고`, `임계 진입` 순간을 시각적으로 보여주기 좋다.

### 4.4 이벤트 영역
최근 이벤트 로그를 시간순으로 표시한다.

권장 이벤트:
- `WARNING`
- `RECOVERY`
- `COOLING_COMMAND_SENT`
- `COOLING_COMMAND_SKIPPED`
- `VIBRATION_WARNING`
- `VIBRATION_CRITICAL`
- `SHUTDOWN`

표시 필드:
- 시각
- 이벤트명
- 심각도
- 간단한 사유

### 4.5 운영지원 영역
규칙 기반 결과와 AI 결과를 분리해 표시한다.

- `규칙 기반 판단`
- `로컬 AI 요약`
- `외부 멀티에이전트 권고`

권장 이유:
- 운영자가 `자동 제어 결과`와 `운영지원 제안`을 혼동하지 않게 하기 위함이다.

## 5. 상태 표현 기준
### 5.1 색상 기준
- `NORMAL`: 녹색
- `WARNING`: 주황색 또는 노란색
- `CRITICAL / SHUTDOWN`: 빨간색

### 5.2 텍스트 기준
- 상태는 영어 코드만 쓰지 않고 한국어 설명을 함께 표시한다.

예:
- `NORMAL / 정상`
- `WARNING / 주의`
- `SHUTDOWN / 정지`

## 6. 핵심 시나리오별 표시 포인트
### 6.1 정상 시나리오
- 온도와 진동이 정상 범위에 있음
- 최근 이벤트 없음 또는 정상 유지
- 냉각 제어 명령 상태 `OFF`

### 6.2 온도 회복 시나리오
- 온도 상승 후 냉각 제어 명령 상태 `ON`
- 이후 온도 하강 추세 표시
- 최근 이벤트에 `WARNING`, `COOLING_COMMAND_SENT`, `RECOVERY` 표시

### 6.3 냉각 실패 시나리오
- 온도 상승 후 냉각 제어 명령 상태 `NOT_SENT`
- 온도 비회복 및 상승 추세 표시
- 최근 이벤트에 `WARNING`, `COOLING_COMMAND_SKIPPED`, `SHUTDOWN` 표시

### 6.4 진동 경고 시나리오
- 진동 차트가 경고 구간으로 상승
- 이벤트에 `VIBRATION_WARNING` 표시
- AI 운영지원 영역에 점검 가이드 표시

### 6.5 진동 셧다운 시나리오
- 진동이 임계 구간 도달
- 설비 상태 `SHUTDOWN`
- 이벤트에 `VIBRATION_CRITICAL`, `SHUTDOWN` 표시

## 7. 메시지스키마와의 연결
운영 대시보드는 아래 객체를 기준으로 화면을 구성한다.

- `latestSensor`
- `lastControl`
- `lastEvent`
- `ruleDecision`
- `localAiDecision`
- `agentDecision`

즉, 대시보드는 원시 MQTT(Message Queuing Telemetry Transport) 메시지를 직접 보여주기보다, 엣지제어운영서버가 만든 통합 상태 객체를 중심으로 동작하는 것이 적절하다.

권장 초기 매핑:
- 상단 상태 카드 -> `runtimeState`, `healthState`, `alertLevel`
- 센서 카드와 차트 -> `latestSensor`
- 최근 냉각 제어 명령 상태 -> `lastControl`
- 최근 이벤트 로그 -> `lastEvent`
- 규칙 기반 판단 -> `ruleDecision`
- AI 결과 -> `localAiDecision`, `agentDecision`

## 8. 구현 권장사항
- 상단 요약 카드와 하단 상세 영역을 분리한다.
- 센서 차트는 2개 이상을 한 그래프에 무리하게 합치지 않는다.
- 셧다운 상태는 시각적으로 가장 강하게 표시한다.
- AI 결과가 없을 때는 빈칸 대신 `결과 없음` 또는 `호출 대기`를 표시한다.
- 초기 MVP에서는 수동 제어 버튼보다 상태 표시와 시나리오 설명에 집중한다.

## 9. 고도화 항목
아래 항목은 초기 MVP 이후 확장 대상으로 본다.

- 에어컨 실제 응답 상태 표시
- 응답 timeout 기반 냉각 실패 경고
- 복수 설비/복수 룸 대시보드
- 3D 디지털트윈 시각화 또는 3D 메타버스 시각화 연동
