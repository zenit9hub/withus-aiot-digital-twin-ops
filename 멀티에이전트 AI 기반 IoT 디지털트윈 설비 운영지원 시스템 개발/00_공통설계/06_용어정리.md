# 용어정리

## 1. 문서 목적
본 문서는 프로젝트 전반에서 반복 사용하는 핵심 용어를 통일하기 위한 기준 문서다.  
목적은 상위 문서, 상세설계 문서, 구현 문서에서 같은 대상을 같은 이름으로 표현하도록 맞추는 데 있다.

## 2. 기본 원칙
- 상위 문서에서는 목적과 역할이 드러나는 표현을 우선 사용한다.
- 상세설계 문서에서는 구현에 필요한 코드성 용어를 함께 사용한다.
- 한국어 설명과 영어 코드 용어가 함께 필요한 경우, 한국어 설명을 먼저 두고 코드 용어를 보조적으로 사용한다.

## 3. 시스템 구성 용어
| 권장 용어 | 대체/비권장 표현 | 사용 원칙 |
| --- | --- | --- |
| 설비환경시뮬레이터 | 가상 센서 디바이스 | 공식 문서에서는 `설비환경시뮬레이터`를 기본으로 사용한다. |
| 엣지제어운영서버 | 엣지 서버 | 공식 문서에서는 제어와 운영 역할이 드러나는 `엣지제어운영서버`를 사용한다. |
| 운영 대시보드 | 대시보드, Dashboard | 설명 문장에서는 `운영 대시보드`, 파일명이나 제품명 맥락에서는 `Dashboard`를 사용할 수 있다. |
| 로컬 AI | 로컬 LLM | 상위 문서에서는 `로컬 AI`, 구현 문맥에서는 `로컬 LLM`을 병기할 수 있다. |
| 외부 멀티에이전트 API | 멀티에이전트 API | 상위 문서에서는 `외부`를 포함해 경계를 분명히 한다. |

## 4. 제어 및 시나리오 용어
| 권장 용어 | 대체/비권장 표현 | 사용 원칙 |
| --- | --- | --- |
| 냉각 제어 | 에어컨 제어 | 시스템 목적을 설명할 때는 `냉각 제어`, 토픽이나 디바이스 명은 `aircon`을 사용한다. |
| 냉각 제어 메시지 | 에어컨 ON 메시지 | 문서에서는 목적이 보이는 `냉각 제어 메시지`를 기본으로 사용한다. |
| 최근 냉각 제어 명령 상태 | 에어컨 상태 | MVP에서는 실제 응답 상태가 아니라 `명령 상태`를 보여준다는 점을 분명히 한다. |
| 냉각 실패 시나리오 | 에어컨 실패 | 초기 MVP에서는 `제어 메시지 미전달 시나리오` 의미로 사용한다. |
| 냉각 제어 미전달 | 제어 미전달 | 시나리오 설명에서는 무엇이 미전달되었는지 명확히 적는다. |

## 5. 시뮬레이터 관련 용어
| 권장 용어 | 대체/비권장 표현 | 사용 원칙 |
| --- | --- | --- |
| 가상 에어컨 장치 | 에어컨 장치 | 시뮬레이터 내부의 IoT(Internet of Things) 디바이스 연동 대상을 설명할 때 사용한다. |
| 가상 에어컨 장치 원격 제어 메시지 | 냉각 제어 명령 | Tick 문서처럼 구현 관점에서 장치 연동을 강조할 때 사용한다. |
| 환경 반응 | 실패 해석 | 시뮬레이터는 `환경 반응`만 담당하고 `실패 해석`은 하지 않는다는 점을 유지한다. |
| 설비 인접 온도 센서 | 룸 온도 센서 | 초기 MVP에서는 `설비 인접 온도 센서`를 사용한다. |

## 6. 데이터 및 상태 용어
| 권장 용어 | 대체/비권장 표현 | 사용 원칙 |
| --- | --- | --- |
| sampleId | snapshotId와 혼용 | MQTT(Message Queuing Telemetry Transport) 메시지 간 동일 시점 연결 키는 `sampleId`를 기본으로 사용한다. |
| snapshotId | sampleId와 혼용 | AI 입력처럼 조합 완료 객체를 설명할 때만 사용할 수 있다. |
| runtimeState | running state | 설비의 현재 동작 상태를 표현할 때 사용한다. |
| healthState | status | 상태 판정 결과를 구조적으로 표현할 때 사용한다. |
| alertLevel | severity와 혼용 | 현재 경고 수준을 표현할 때 사용한다. |

## 7. 이벤트 용어
초기 MVP의 권장 이벤트명은 아래와 같다.

- `WARNING`
- `RECOVERY`
- `COOLING_COMMAND_SENT`
- `COOLING_COMMAND_SKIPPED`
- `VIBRATION_WARNING`
- `VIBRATION_CRITICAL`
- `SHUTDOWN`

아래 표현은 초기 MVP의 공식 이벤트명으로 사용하지 않는다.

- `AIRCON_ON_REQUESTED`
- `AIRCON_ON_CONFIRMED`
- `COOLING_FAILURE`

## 8. 판단 출처 용어
판단 출처는 아래 코드로 통일한다.

- `RULE`
- `LOCAL_AI`
- `AGENT_API`

설명 문장에서는 아래처럼 병기할 수 있다.

- `RULE(규칙 기반 제어)`
- `LOCAL_AI(로컬 AI 결과)`
- `AGENT_API(외부 멀티에이전트 결과)`

## 9. 고도화 항목에서만 사용하는 용어
아래 용어는 초기 MVP 핵심 문서에서는 보조적으로만 사용하고, 고도화 항목에서 본격적으로 다룬다.

- 에어컨 `status`
- 응답 확인
- timeout
- 냉각 미회복 경고
- 제어 실패 상세 메시지
