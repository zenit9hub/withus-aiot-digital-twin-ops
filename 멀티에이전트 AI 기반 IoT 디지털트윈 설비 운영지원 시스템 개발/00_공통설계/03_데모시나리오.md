# 데모시나리오

## 1. 문서 목적
본 문서는 핵심본체 샘플 프로젝트의 시연 흐름을 교육용으로 정리한 문서다.  
목적은 `센서 수집 -> 상태 판정 -> 제어 -> AI(Artificial Intelligence) 운영지원 -> 대시보드 표시` 흐름을 짧고 명확하게 설명하는 데 있다.

## 2. 시나리오 설계 원칙
- `가상 센서 디바이스`는 실제로는 `설비/환경 시뮬레이터` 역할을 한다.
- 시뮬레이터는 온도와 진동을 내부에서 함께 계산하되, 외부 MQTT(Message Queuing Telemetry Transport) 인터페이스는 센서별 토픽으로 분리한다.
- 시뮬레이터는 실패를 해석하지 않고, 냉각 제어 메시지 수신 여부에 따라 냉각 효과를 반영한다.
- 핵심본체 MVP(Minimum Viable Product)에서는 에어컨 status 확인이나 실패 판정을 수행하지 않는다.
- 응답 없음, 냉각 미회복 경고는 고도화 항목으로 다룬다.

## 3. 구성요소 역할
### 3.1 설비환경시뮬레이터(Node.js)
- 컨베이어 상태, 온도, 진동을 계산한다.
- 온도 센서, 진동 센서, 설비 상태를 발행한다.
- 냉각 제어 메시지를 구독한다.

### 3.2 엣지제어운영서버(Node-RED/FlowFuse)
- 센서 데이터를 수집하고 상태를 판정한다.
- 시나리오에 따라 냉각 제어 메시지를 전달하거나 전달하지 않는다.
- 셧다운을 수행한다.
- 이벤트와 AI 입력 요약 객체를 생성한다.
- 대시보드에 상태와 결과를 표시한다.

### 3.3 로컬 AI / 외부 멀티에이전트
- 원인 후보를 제시한다.
- 운영 가이드를 제시한다.
- 최종 운영 권고를 생성한다.

## 4. 제어 기준
### 4.1 온도 기준
- `45도 이상`: WARNING, 냉각 제어 수행
- `50도 이상`: CRITICAL
- `50도 이상이 일정 시간 지속`: SHUTDOWN

### 4.2 진동 기준
- `0.70 이상`: VIBRATION_WARNING
- `0.85 이상`: VIBRATION_CRITICAL, 즉시 SHUTDOWN

## 5. 핵심 시나리오
### 시나리오 A. 정상 상태
목적: 기본 데이터 흐름과 대시보드 기본 상태 확인

- 시뮬레이터가 정상 온도와 정상 진동 데이터를 발행한다.
- 엣지제어운영서버는 정상 상태로 판정한다.
- 대시보드에는 `정상`, `경보 없음`이 표시된다.

### 시나리오 B. 온도 경고 후 냉각 제어 메시지 전달로 회복
목적: 냉각 제어와 회복 흐름 확인

- 온도가 45도 이상으로 상승한다.
- 엣지제어운영서버가 WARNING 상태로 판정하고 냉각 제어 메시지를 시뮬레이터로 전송한다.
- 시뮬레이터가 냉각 효과를 반영한다.
- 온도가 서서히 하강하고 정상 범위로 복귀한다.
- 대시보드에는 회복 상태와 운영지원 요약이 표시된다.

### 시나리오 C. 온도 경고 후 냉각 제어 메시지 미전달로 셧다운
목적: 시나리오에 따른 결과 차이 설명

- 온도가 45도 이상으로 상승한다.
- 냉각 실패 시나리오에서는 시뮬레이터로 냉각 제어 메시지가 전송되지 않는다.
- 냉각 효과가 반영되지 않으므로 온도는 계속 상승한다.
- 온도가 50도 이상으로 일정 시간 유지되면 규칙 기반 제어가 셧다운을 수행한다.
- 대시보드에는 `냉각 제어 미전달`, `고온 지속`, `셧다운` 흐름이 표시된다.
- AI 운영지원은 냉각 제어 미적용 상태에서 과열이 지속되었다는 요약을 제시할 수 있다.

### 시나리오 D. 진동 경고 후 운영지원 표시
목적: 셧다운 전 단계의 운영지원 흐름 설명

- 진동값이 경고 구간으로 상승한다.
- 엣지제어운영서버는 `VIBRATION_WARNING`으로 판정한다.
- 즉시 셧다운은 수행하지 않는다.
- 대시보드에는 경고 상태와 운영지원 권고가 표시된다.

### 시나리오 E. 진동 임계치 초과로 즉시 셧다운
목적: 기계적 위험 상황에서의 즉시 보호 동작 설명

- 진동값이 임계치 이상으로 상승한다.
- 엣지제어운영서버는 `VIBRATION_CRITICAL`로 판정한다.
- 규칙 기반 제어가 즉시 셧다운을 수행한다.
- 대시보드에는 셧다운 상태와 AI 권고가 표시된다.

## 6. 상태 전이 요약
- `NORMAL -> TEMP_WARNING -> COOLING_ACTIVE -> RECOVERED`
- `NORMAL -> TEMP_WARNING -> TEMP_CRITICAL -> SHUTDOWN`
- `NORMAL -> VIBRATION_WARNING`
- `VIBRATION_WARNING -> VIBRATION_CRITICAL -> SHUTDOWN`

## 7. 대시보드 표시 항목
- 현재 온도
- 현재 진동
- 설비 상태
- 최근 냉각 제어 명령 상태
- 최근 이벤트 로그
- 셧다운 여부
- 로컬 AI 요약
- 외부 멀티에이전트 권고
- 판단 출처

## 8. 구현상 권장사항
- 시뮬레이터는 환경 상태 계산에 집중하고, 실패 해석은 하지 않는다.
- 정상/냉각 실패 시나리오는 제어 메시지 전달 여부로 구분한다.
- 응답 없음, 냉각 미회복 메시지는 고도화 항목으로 분리한다.
- 센서 토픽이 분리되므로, 같은 시점의 데이터를 조합할 수 있어야 한다.
