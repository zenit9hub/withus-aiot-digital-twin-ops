# 메시지스키마

## 1. 문서 목적
본 문서는 핵심본체 샘플 프로젝트에서 사용하는 센서, 상태, 제어, AI(Artificial Intelligence) 입력/출력 메시지 구조를 정의한다.  
목적은 설비환경시뮬레이터, 엣지제어운영서버, 로컬 AI, 외부 멀티에이전트 API(Application Programming Interface), 대시보드가 같은 데이터 기준을 사용하도록 맞추는 데 있다.

## 2. 설계 원칙
- MQTT(Message Queuing Telemetry Transport) 외부 인터페이스는 단순하게 유지한다.
- 센서는 `<sensorId>/sensor`, 설비와 액추에이터는 `<assetId>/status`, `<assetId>/control` 패턴을 사용한다.
- 이벤트는 기본적으로 엣지제어운영서버 내부에서 생성한다.
- 온도와 진동은 서로 다른 토픽으로 발행하되, 같은 시점 데이터는 `sampleId`로 연결한다.
- 핵심본체 MVP에서는 에어컨 status 기반 실패 판정을 수행하지 않는다.

## 3. MQTT 토픽 구조
### 3.1 핵심 토픽
- `factory/room-01/conveyor-01/status`
- `factory/room-01/conveyor-01/control`
- `factory/room-01/temp-01/sensor`
- `factory/room-01/vib-01/sensor`
- `factory/room-01/aircon-01/control`

### 3.2 선택 확장 토픽
- `factory/room-01/aircon-01/status`

## 4. 공통 필드
- `timestamp`: ISO 8601(국제표준 시각 형식) 시각
- `sampleId`: 같은 시점 데이터를 묶는 식별자
- `roomId`: 룸 식별자
- `equipmentId` 또는 `sensorId`: 자산 식별자

## 6. 이벤트 객체
이벤트는 기본적으로 엣지제어운영서버 내부에서 생성한다.

권장 필드:
- `timestamp`
- `equipmentId`
- `eventType`
- `severity`
- `reason`

권장 `eventType`:
- `WARNING`
- `SHUTDOWN`
- `RECOVERY`
- `COOLING_COMMAND_SENT`
- `COOLING_COMMAND_SKIPPED`
- `VIBRATION_WARNING`
- `VIBRATION_CRITICAL`

## 7. AI 입력 객체
AI 입력은 원시 센서 전체가 아니라 요약 정보만 포함한다.

필수 필드:
- `timestamp`
- `snapshotId`
- `equipmentId`
- `roomId`
- `currentStatus`
- `runtimeState`
- `shutdownFlag`
- `alertLevel`
- `recentSensorSummary`
- `controlSummary`
- `trendSummary`
- `thresholdEvents`

JSON 예시:
```json

  "timestamp": "2026-04-04T10:31:20+09:00",
  "snapshotId": "snapshot-000123",
  "equipmentId": "conveyor-01",
  "roomId": "room-01",
  "currentStatus": "SHUTDOWN",
  "runtimeState": "SHUTDOWN",
  "shutdownFlag": true,
  "alertLevel": "CRITICAL",
  "recentSensorSummary": {
    "temperatureAvg": 48.9,
    "temperatureMax": 50.6,
```

## 10. 초기 판정 기준
### 온도
- `temperature < 45`: `NORMAL`
- `45 <= temperature < 50`: `WARNING`, 냉각 제어 수행
- `temperature >= 50`: `CRITICAL`, 조건 지속 시 `SHUTDOWN`

### 진동
- `vibration < 0.70`: `NORMAL`
- `0.70 <= vibration < 0.85`: `VIBRATION_WARNING`
- `vibration >= 0.85`: `VIBRATION_CRITICAL`, 즉시 `SHUTDOWN`

## 11. 구현 메모
- 시뮬레이터는 내부적으로 풍부한 상태를 유지하고, 외부에는 최소 토픽만 노출한다.
- 외부 온도 센서는 `설비 인접 온도 센서` 1개만 사용한다.
- `ambientBaselineTemp`는 시뮬레이터 내부 기준값으로만 유지한다.
- 센서 토픽이 분리되므로, 엣지제어운영서버는 같은 시점의 데이터를 조합할 수 있어야 한다.
- 에어컨 `status` 기반 실패 판정은 고도화 항목으로 분리한다.
