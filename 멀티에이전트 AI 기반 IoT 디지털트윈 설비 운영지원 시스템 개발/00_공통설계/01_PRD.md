# PRD(Product Requirements Document)

## 1. 문서 목적
본 문서는 `멀티에이전트 AI(Artificial Intelligence) 기반 IoT(Internet of Things) 디지털트윈 설비 운영지원 시스템 개발`의 핵심본체 요구사항을 정리한 상위 기준 문서다.  
이 문서는 상세설계보다 먼저 읽는 문서이며, 프로젝트의 목표, 범위, 구성, 완료 기준을 정의한다.

## 2. 프로젝트 정의
이 프로젝트는 `설비환경시뮬레이터`, `엣지제어운영서버`, `로컬 AI`, `외부 멀티에이전트 API(Application Programming Interface)`를 연결하여, 설비 상태를 감지하고 운영 의사결정을 지원하는 교육용 레퍼런스 프로젝트다.

### 핵심 학습 포인트
- 센서, 제어, 운영지원, 대시보드가 연결되는 전체 흐름을 이해한다.
- `규칙 기반 안전 제어`와 `AI 기반 운영지원`의 역할 차이를 구분한다.
- 단일 설비 기준의 핵심본체를 완성한 뒤, 실제 설비 연동이나 복수 설비 운영으로 확장하는 구조를 이해한다.

## 3. 목표
- 온도, 진동, 냉각 제어, 셧다운, 운영지원이 하나의 데모 흐름으로 연결되는 전체 흐름 샘플을 만든다.
- 학생과 멘토가 공통 기준으로 활용할 수 있는 재사용 가능한 레퍼런스 구조를 만든다.
- 설명하기 쉬운 구조를 유지하면서, 이후 실제 설비 연동과 복수 설비 운영으로 확장 가능한 기반을 확보한다.

## 4. 범위
### 4.1 핵심본체 범위
- 설비 1종, 룸 1개, 에어컨 1개 기준의 단일 시뮬레이션 환경
- 설비 인접 온도 센서 1개, 진동 센서 1개
- MQTT(Message Queuing Telemetry Transport) 기반 센서, 상태, 제어 토픽 연계
- 엣지 수집 및 제어 서버 기반 상태 판정과 운영 대시보드
- 로컬 LLM(Large Language Model) 기반 운영지원
- 외부 멀티에이전트 API 기반 운영지원

### 4.2 고도화 범위
- 3D 디지털트윈 시각화 또는 3D 메타버스 시각화
- 에어컨 status 기반 응답 확인 및 냉각 실패 판정
- 실제 설비 및 실제 센서 연동
- 복수 설비 및 복수 룸 동시 운영
- 운영 환경 수준의 인증, 권한, 장애복구 체계

### 4.3 제외 범위
- 고정밀 CAD 수준 3D 모델링 및 물리 기반 시뮬레이션

## 5. 시스템 구성
`설비환경시뮬레이터 -> MQTT -> 엣지제어운영서버 -> 로컬 AI / 외부 멀티에이전트 API -> 운영 대시보드`

- 설비환경시뮬레이터: 설비 상태, 온도, 진동을 계산하고, 냉각 제어 메시지 수신 시 냉각 효과를 반영한다.
- 엣지제어운영서버: 데이터를 수집하고, 상태를 판정하며, 시나리오에 따라 냉각 제어 메시지를 전달하고 셧다운을 수행한다.
- 로컬 AI: 상태 요약, 원인 후보, 점검 가이드를 생성한다.
- 외부 멀티에이전트 API: 역할 분담 기반 운영지원 판단을 생성한다.
- 운영 대시보드: 상태, 경보, 제어 결과, 운영지원 결과를 표시한다.

## 6. 핵심 원칙
- 셧다운과 같은 안전 제어는 규칙 기반 자동화 로직이 담당한다.
- AI는 제어가 아니라 운영지원 판단을 담당한다.
- 시뮬레이터는 실패를 해석하지 않고, 제어 메시지 수신 여부에 따라 환경 변화를 반영한다.
- 핵심본체 MVP(Minimum Viable Product)에서는 에어컨 status 기반 실패 판정을 수행하지 않는다.
- 응답 확인, timeout, 냉각 미회복 경고는 고도화 항목으로 분리한다.

## 7. 핵심 요구사항
### 7.1 설비환경시뮬레이터
- 온도와 진동을 하나의 시뮬레이터 내부에서 함께 계산해야 한다.
- 외부 MQTT 인터페이스는 온도 센서 토픽과 진동 센서 토픽으로 분리해야 한다.
- 냉각 제어 메시지를 수신하면 냉각 효과를 반영해야 한다.
- 장비 OFF 시 온도는 상온 기준 방향으로 복귀해야 한다.

### 7.2 엣지제어운영서버
- 온도 경고 구간 진입 시 냉각 제어를 수행해야 한다.
- 정상 시나리오와 냉각 실패 시나리오를 재현할 수 있어야 한다.
- 온도 위험 구간 지속 시 셧다운을 수행해야 한다.
- 진동 임계치 초과 시 즉시 셧다운을 수행해야 한다.
- 상태, 경보, 제어 결과, AI 결과를 운영 대시보드에 반영해야 한다.

### 7.3 AI 운영지원
- 로컬 AI는 최근 상태 요약을 받아 원인 후보와 점검 가이드를 생성해야 한다.
- 외부 멀티에이전트 API는 역할 분리형 운영지원 구조를 통해 결과를 생성해야 한다.
- 최종 운영지원 결과는 운영 대시보드에 표시되어야 한다.

## 8. 핵심 데모 시나리오
- 정상 상태
- 온도 경고 후 냉각 제어 메시지 전달로 회복
- 온도 경고 후 냉각 제어 메시지 미전달로 셧다운
- 진동 경고 후 운영지원 알림
- 진동 임계치 초과로 즉시 셧다운

## 9. 완료 기준
- 센서 데이터와 설비 상태가 운영 대시보드에 실시간 반영된다.
- 정상 시나리오와 냉각 제어 메시지 미전달 시나리오가 모두 재현된다.
- 진동 경고와 진동 셧다운 시나리오가 재현된다.
- 로컬 AI와 외부 멀티에이전트 결과가 표시된다.
- 규칙 기반 안전 제어와 AI 기반 운영지원의 역할 차이를 설명할 수 있다.
