# 아키텍처_설비환경시뮬레이터

## 1. 문서 목적
본 문서는 `설비환경시뮬레이터`의 역할, 제어 메시지 수신 구조, tick 기반 환경 계산 구조를 함께 정의한다.

## 2. 역할
설비환경시뮬레이터는 컨베이어, 가상 에어컨 장치, 온도, 진동이 포함된 가상 환경을 계산하고 발행하는 구성요소다.

핵심 역할:
- 설비 상태 계산
- 환경 상태 계산
- IoT(Internet of Things) 제어 메시지 수신
- 상태값과 센서값 발행

## 3. 원격 제어 대상
시뮬레이터 환경 안에서 IoT 원격 제어 대상은 아래 2개다.

- 컨베이어 벨트
- 에어컨

두 장치는 모두 MQTT(Message Queuing Telemetry Transport) `~/control` 토픽을 구독한다.

## 4. MQTT 인터페이스
### 4.1 구독 토픽
- `factory/room-01/conveyor-01/control`
- `factory/room-01/aircon-01/control`

### 4.2 발행 토픽
- `factory/room-01/conveyor-01/status`
- `factory/room-01/temp-01/sensor`
- `factory/room-01/vib-01/sensor`

## 5. 내부 상태
초기 MVP(Minimum Viable Product)에서 필요한 내부 상태는 아래 정도면 충분하다.

- `conveyorRunning`
- `coolingCommandActive`
- `equipmentTemperature`
- `vibration`
- `ambientBaselineTemp`
- `airconTargetTemp`
- `sampleId`

## 6. 초기 구현 상수
초기 구현은 아래 정도의 상수로 시작하면 충분하다.

- `TICK_MS = 1000`
- `ambientBaselineTemp = 25`
- `airconTargetTemp = 22`
- `heatGainPerTick = 0.8`
- `coolingGainPerTick = 1.0`

위 값은 구현 시작용 예시이며, 데모 과정에서 조정할 수 있다.

## 7. 제어 메시지 수신부
제어 메시지 수신부는 MQTT `control` 토픽을 구독하고, 내부 상태를 변경한다.

예:
- `conveyor/control` 수신 -> `conveyorRunning` 변경
- `aircon/control` 수신 -> `coolingCommandActive` 변경

즉, 제어 메시지 수신부는 `입력 처리`, tick 루프는 `환경 계산`을 담당한다.

권장 payload 예시:

`conveyor/control`
```json
{
  "timestamp": "2026-04-05T10:00:00+09:00",
  "equipmentId": "conveyor-01",
  "roomId": "room-01",
  "controlAction": "RESET",
  "controlSource": "RULE",
  "reason": "start simulation"
}
```

`aircon/control`
```json
{
  "timestamp": "2026-04-05T10:00:05+09:00",
  "roomId": "room-01",
  "airconId": "aircon-01",
  "command": "ON",
  "controlSource": "RULE"
}
```

## 8. Tick 기반 환경 계산

### 8.1 Tick 처리 순서
1. 현재 내부 상태를 읽는다.
2. 컨베이어 상태에 따라 온도를 계산한다.
3. 에어컨 상태에 따라 냉각 효과를 계산한다.
4. 진동을 계산한다.
5. `sampleId`를 생성한다.
6. 상태값과 센서값을 발행한다.

### 8.2 Tick 발행 항목
매 tick에서 아래 메시지를 발행한다.

- `conveyor/status`
- `temp/sensor`
- `vib/sensor`

같은 tick에서 발행하는 메시지는 동일한 `sampleId`를 사용한다.

## 9. 동작 원칙
- 컨베이어 ON이면 온도는 상승한다.
- 에어컨 제어 메시지가 수신되면 냉각 효과를 반영한다.
- 에어컨 제어 메시지가 수신되지 않으면 냉각 효과를 반영하지 않는다.
- 컨베이어와 에어컨이 모두 OFF면 온도는 기준 상온 방향으로 복귀한다.
- 진동은 단순 센서값으로만 계산·발행하며, 정상/경고/위험 판정은 시뮬레이터 범위에 포함하지 않는다.

## 10. 책임 경계
- 설비환경시뮬레이터는 환경 상태를 계산하고 발행한다.
- 엣지제어운영서버는 상태를 해석하고 제어를 결정한다.
- 시뮬레이터는 실패를 해석하거나 셧다운을 결정하지 않는다.
